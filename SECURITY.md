# Security Policy

## Unterstützte Versionen

Sicherheitskorrekturen werden grundsätzlich nur für die aktuelle Version vorbereitet.

## Sicherheitslücken melden

Bitte keine sicherheitskritischen Details in einem öffentlichen Issue veröffentlichen. Verwende stattdessen die Funktion **Security → Report a vulnerability** des GitHub-Repositories und erstelle eine private Security Advisory.

Füge nach Möglichkeit folgende Angaben bei:

- betroffene EEDTOY-Version
- Betriebssystem
- reproduzierbare Schritte
- erwartetes und tatsächliches Verhalten
- relevante Logauszüge ohne persönliche Zugangsdaten
